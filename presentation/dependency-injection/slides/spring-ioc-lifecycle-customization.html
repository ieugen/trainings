Bean Life Cycle (3)

  • Bean lifecycle consists of call back methods
  • There are two broad groups:
    • post initialization call backs
    • pre destruction call backs
  • There are 4 ways for controlling lifecycle events
    • InitializingBean and DisposableBean interfaces
    • custom init() and destroy() methods in configuration
    • @PostConstruct and @PreDestroy
    • other *Aware interfaces (ServletContextAware) (**)
  • you can set default names for custom init and destroy methods
