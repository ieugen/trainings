Using scoped beans as dependencies

Scoped beans other then singleton and prototype require an AOP proxy 

<bean id="userPreferences" class="com.foo.UserPreferences"
    scope="session">
    <!-- instructs the container to proxy the surrounding bean -->
    <aop:scoped-proxy/>
    </bean>
    <bean id="userService" class="com.foo.SimpleUserService">
    <property name="userPreferences" ref="userPreferences"/>
    </bean>

HTTP Session-scoped bean exposed as a proxy ensures one UserPreferences per Session
