Spring Configuration Metadata

Configuration meta-data can be supplied using

  • XML format
  • Java-based configuration, since spring 3.0
  • Annotation-based configuration, since Spring 2.5
  • A mix of them

Go to spring-configuration module

Discussion: which one to use?
The configuration metadata is represented in
  • XML
  • Java annotations
  • Java code
Discuție: care configurație să fie folosită? Răspuns: Depinde de gustul programatorului.

JavaConfig - este static typed și poate fi validată la compilare mai ușor dar necesită recompilare dacă vreți să modificați configurația.

Configurația XML poate fi generată și validată cu uneltele XML existente. Permite definirea de name-space-uri.

Adnotările sunt invazive (te leagă de Spring sau javax.inject) și nu pot fi aplicate codului legacy.
