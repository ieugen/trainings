Annotation driven configuration

Makes use of annotations to provide configuration metadata

  • are intrusive - pollute code
  • can be overridden by XML configuration
  • configuration is spread across classes - code harder to understand
  • faster development
You can define beans only via @Component or derivatives. Other annotations just provide configuration metadata for bean creation, bot not the BeanDefinition blueprint.
Dacă nu folosiți @Component va trebui să definiți Bean-urile în XML sau Java
